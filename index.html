HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Meta elements are invisible elements used by the webpage and search engines
  • HTML elements are stored in brackets like these: <>

Common HTML Elements

  • h1-6 defines the header. These should be used in order, and generally h1 is used only once
  • ul and ol define unordered lists and ordered lists

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • A border is a box drawn around a selection
  • A shadow can then be added with the box shadow property, which is then defined by its offset
  • A period preceding a selector designates it as a class selector

Git

  • cd file directory: navigates to a specific directory
  • cd: navigates one step upward in the directory tree
  • pwd: prints the working directory
  • git status: prints the current working branch
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout branch-name: switches working branch
  • git pull origin main/master: pulls files from a branch into your working branch
  • git commit: commits current changes to working branch
  • git commit -m "message": commits current changes to working branch with an added message. This should be used every time.
  • git push origin branch-name: creates a pull request from the specified branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A function is a set of instructions that allow a computer to perform a specified task
  • A function must be defined in order to be called into action, but it cannot execute without being called
  • Good use of the console.log command makes your site clearer and easier to code.
